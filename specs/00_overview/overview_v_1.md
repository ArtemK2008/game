# Incremental World-Progression Autobattler — Overview v1

## Core pitch
A grind-heavy incremental game built around **world progression through connected map nodes**, **lazy autobattle combat**, **long-term account growth**, and **small but meaningful build decisions**.

The player advances through regions made of interconnected nodes, unlocks new paths by clearing enough enemies, develops a small roster, improves permanent systems, and revisits older locations for resources and progression. Combat mostly runs itself, while the player mainly interacts through preparation, route choices, upgrade priorities, and long-term optimization.

The game is designed to feel **lazy, grindy, and satisfying**, with progress coming from gradual account growth rather than intense manual play.

---

## What the game is
This is **not** a tower defense game.

It is a **world-progression autobattler** with:
- IdleOn-like forward map progression
- lazy survivor-style combat presentation
- optional level-up choices during active progression
- permanent multi-system growth
- node-based world structure with route choices
- farming of both new and old locations

The core fantasy is:
**enter a zone, grind it down, unlock the next path, grow stronger, and keep pushing deeper into the world.**

---

## Design principles
- The game should feel **grindy on purpose**.
- The game should be **lazy-friendly** and comfortable for long sessions.
- Progress should feel **earned**, not freely handed out.
- Older maps should remain **useful and relevant**.
- Permanent progression should matter more than short-term run success.
- Combat should require minimal manual control.
- Player decisions should focus on **build planning, upgrade priorities, route choices, and map progression**, not action execution.
- Unlocks should remain **permanent**.
- The world should feel like a **progression structure**, not just a list of disconnected stages.

---

## Core gameplay loop
1. Choose the next available node on the world map.
2. Enter the node and let the character fight automatically.
3. During early progression, make upgrade choices manually.
4. Kill enough enemies to progress the node’s unlock meter.
5. Clear the node and unlock the next part of the route.
6. Visit service, loot, elite, or progression nodes depending on route choices.
7. Return to older maps when resources, mastery, or safer farming are needed.
8. Spend materials on permanent upgrades.
9. Repeat with stronger builds, better farming, and deeper progression.

---

## World progression
The world is made of **regions**, and each region contains a chain or small network of **map nodes**.

Each node is a progression step, not just a background.
The player moves forward by **clearing** nodes through repeated combat runs.

### Core world idea
- Regions are persistent parts of the world.
- Each region contains multiple nodes with different purposes.
- Clearing nodes unlocks new paths deeper into the region.
- Some paths may branch, giving the player limited route choice.
- Older regions remain useful because they provide unique farming value.

This keeps progression more interesting than a simple one-line stage list while still preserving a lazy and readable structure.

---

## What opens the next map
The next node unlocks when the player kills enough enemies on the current one.

This can be presented as:
- zone clearing
- threat reduction
- stabilization progress
- expedition completion

Mechanically, the core idea stays simple:
**kill enough enemies on a node to open the path forward.**

This keeps progression intuitive, satisfying, and suitable for lazy/autobattle gameplay.

---

## Node-based map structure
The world should use **different node types**, not just repeated combat maps.

This is one of the main ways to make progression feel more alive and less flat.

### Example node types
- **Combat node** — standard farming and progression node
- **Elite node** — harder combat with better rewards
- **Boss / Gate node** — major regional progression checkpoint
- **Loot node** — higher resource or chest-oriented reward
- **Town / Service node** — rest, upgrades, vendors, utility systems
- **Forge / Progression node** — permanent or semi-permanent build improvement
- **Special event node** — rare or unusual variation in progression

Not every region needs all node types immediately, but the world should be designed around the idea that nodes can serve different roles.

---

## Route choice and route economy
The player should not always move through a completely fixed line.

Limited route choice makes the world more interesting without making it overly complex.

### What route choice can add
- a safer path versus a riskier path
- a resource-focused path versus a progression-focused path
- access to service nodes before harder encounters
- choices about when to push forward and when to stabilize a build

### What route economy means here
As the player progresses through nodes, there should be a rhythm of:
- fighting
- collecting rewards
- deciding whether to spend or save
- hitting service or upgrade nodes
- preparing for harder progression gates

This adds structure to sessions and makes progression feel more alive.

---

## Map progression meter
Each combat-oriented node has a **progress meter**.

The main source of progress is:
- killing regular enemies
- optionally gaining bonus progress from elites or passive milestones

But the backbone should stay simple:
**enemy kills drive unlock progress.**

This keeps the game close to a satisfying grind loop and avoids overcomplicating progression with too many active objectives.

---

## Combat philosophy
Combat is primarily **automatic**.

The player does **not** need direct manual control over movement or attacks.
The core combat fantasy is not mechanical execution, but **build effectiveness over time**.

The player mostly interacts by:
- choosing what node to run
- selecting a build before combat
- choosing upgrades during active runs
- later enabling automation for those choices too

This creates a game that feels active enough to be engaging, but lazy enough to support grind-heavy progression.

---

## Active mode vs auto mode
A node should have progression stages of interaction.

### Active progression mode
When a node is new, the player engages with it more directly by:
- selecting upgrade choices manually during runs
- observing whether the build is working
- deciding what to improve between runs

This is not manual combat.
It is **manual decision-making**.

### Auto mode
Once a node is sufficiently mastered, the player can automate more of the run:
- auto-pick upgrades
- repeat farming runs
- rely on the build to perform without constant attention

### AFK farming
Fully mastered or sufficiently progressed nodes can become comfortable farming zones for long-term grind.

This creates a good progression rhythm:
- new content requires attention,
- old content becomes efficient and lazy.

---

## Role of level-up choices
Level-up choices still matter, but they are **not the main progression gate**.

Their role is to improve:
- clear speed
- farming efficiency
- build identity
- performance on harder nodes
- route flexibility and confidence in riskier paths

So the game keeps some survivor-like flavor, but that layer is no longer the main structure of the entire concept.

It is a **supporting combat layer**, not the backbone.

---

## Permanent progression
Permanent progression is one of the main pillars of the game.

Players should invest resources into long-term systems such as:
- account-wide upgrades
- character growth
- build unlocks
- new systems
- resource efficiency
- node farming efficiency
- future specialization options

The goal is for every grind session to contribute to something that lasts.

---

## Old maps staying relevant
Older regions and nodes should never become useless.

They remain valuable because they can provide:
- unique regional resources
- safer farming
- faster farming
- mastery progress
- materials needed for future upgrades
- specific build-enabling drops or unlock requirements
- better efficiency for certain node types

This is one of the strongest progression feelings the concept should preserve.

---

## Character and roster direction
The game can still include a **small roster of characters**, but the roster should stay manageable.

Characters mainly exist to:
- provide different farming styles or efficiencies
- support build diversity
- create long-term account goals
- give variety without overwhelming the player

The game should avoid making character management too heavy.
The roster is there to deepen progression, not to turn the game into a party micromanagement system.

---

## Session structure
The game should naturally support short and long sessions.

A good session rhythm looks like:
- clear a combat node
- collect rewards
- choose the next route step
- visit a useful service node if needed
- continue pushing or stop safely after a clear node

This makes the game well-suited for a lazy progression loop because it gives players natural stopping points.

---

## What the concept is moving away from
The concept is intentionally moving away from:
- tower defense as the core genre
- direct manual combat skill expression
- high-pressure action gameplay
- overly complex unlock conditions
- objective design that requires precise manual play
- pure roguelike run identity as the main structure

This version is stronger when it stays focused on:
- lazy grinding
- automatic combat
- node/map unlocking through kills
- permanent growth
- satisfying long-term progression
- route structure without excessive complexity

---

## Genre position
The game is best described as something close to:

**a grind-heavy incremental world-progression autobattler with survivor-style combat flavor**

Or more simply:

**an IdleOn-like progression game with lazy auto-combat, node-based routes, and map-based grinding**

---

## Final summary
This concept is built around progressing through a world of connected nodes by killing enough enemies to unlock the path forward. Combat is mostly automatic, with player input focused on upgrade decisions, build setup, route choices, and long-term progression planning rather than direct control. New nodes ask for attention, while older nodes become comfortable auto-farming zones. Different node types and limited route choices make the world feel more alive, while permanent progression keeps every grind session meaningful. The result is a lazy, grindy progression game where the main satisfaction comes from steady growth, efficient farming, unlocking new locations, and building an account that becomes stronger over time.
